# 智慧健控微信小程序开发文档

## 1. 项目概述

智慧健控是一款专为老弱病残人群设计的微信小程序，提供康复护理和健康保养服务。主要功能包括健康数据监测、康复训练指导、用药提醒、紧急求助和护理知识普及等。

## 2. 环境配置

### 2.1 开发工具

- **微信开发者工具**：v1.06.2309070 或更高版本
- **Node.js**：v14.0.0 或更高版本
- **操作系统**：Windows 7/8/10/11，macOS

### 2.2 微信小程序配置

- **AppID**：需要在微信公众平台注册获取
- **开发模式**：选择"小程序"开发模式
- **项目设置**：
  - 启用 ES6 转 ES5
  - 启用增强编译
  - 启用代码压缩

## 3. 代码结构说明

### 3.1 全局配置

- **app.json**：配置小程序的页面路径、窗口样式、底部导航栏等
- **app.wxss**：定义全局样式，包括字体、颜色、布局等
- **app.js**：小程序的全局逻辑，包括初始化、授权管理、全局数据等

### 3.2 页面结构

每个页面由三个文件组成：
- **.wxml**：页面结构，使用微信小程序的标签语言
- **.wxss**：页面样式，使用类似CSS的语法
- **.js**：页面逻辑，处理用户交互、数据管理等

### 3.3 核心功能模块

#### 3.3.1 健康档案管理
- **页面**：pages/health-record/
- **功能**：个人信息管理、健康状况记录、健康数据记录、健康报告生成
- **数据结构**：
  ```javascript
  userInfo: {
    name: String,      // 姓名
    age: Number,       // 年龄
    gender: String,    // 性别
    phone: String,     // 联系电话
    emergencyContact: String // 紧急联系人
  }
  healthCondition: {
    pastIllnesses: String,      // 既往病史
    allergies: String,          // 过敏史
    currentMedications: String  // 当前用药
  }
  ```

#### 3.3.2 健康数据监测
- **页面**：pages/health-monitor/
- **功能**：血压、血糖、心率数据记录，数据趋势分析，智能设备连接
- **数据结构**：
  ```javascript
  healthData: {
    bloodPressure: [  // 血压记录
      { date: String, time: String, value: String }
    ],
    bloodSugar: [     // 血糖记录
      { date: String, time: String, value: String, type: String }
    ],
    heartRate: [      // 心率记录
      { date: String, time: String, value: String }
    ],
    weight: [         // 体重记录
      { date: String, time: String, value: String }
    ]
  }
  ```

#### 3.3.3 康复训练指导
- **页面**：pages/rehabilitation/
- **功能**：训练计划管理、训练指导、训练记录、视频指导
- **数据结构**：
  ```javascript
  trainingRecords: [
    { date: String, time: String, content: String, duration: String }
  ]
  ```

#### 3.3.4 用药提醒
- **页面**：pages/medication/
- **功能**：今日用药提醒、用药管理、用药记录、用药知识
- **数据结构**：
  ```javascript
  todayMedications: [
    { name: String, time: String, dosage: String, status: String }
  ],
  medicationRecords: [
    { date: String, name: String, time: String, status: String }
  ]
  ```

#### 3.3.5 紧急求助
- **页面**：pages/emergency/
- **功能**：紧急求助按钮、紧急联系人管理、附近医院查询、急救常识
- **数据结构**：
  ```javascript
  emergencyContacts: [
    { name: String, relation: String, phone: String }
  ],
  nearbyHospitals: [
    { name: String, address: String, distance: String }
  ]
  ```

#### 3.3.6 护理知识普及
- **页面**：pages/knowledge/
- **功能**：知识分类、知识列表、健康资讯、常见问题
- **数据结构**：
  ```javascript
  knowledgeList: [
    { title: String, summary: String, date: String }
  ],
  healthNews: [
    { title: String, date: String }
  ],
  faqList: [
    { question: String, answer: String }
  ]
  ```

## 4. 技术实现

### 4.1 前端技术

- **微信小程序原生框架**：使用微信官方提供的框架进行开发
- **WXML**：用于构建页面结构
- **WXSS**：用于样式设计
- **JavaScript**：用于逻辑处理
- **Flexbox**：用于布局
- **CSS动画**：用于交互效果

### 4.2 数据管理

- **全局数据**：使用app.js中的globalData存储全局数据
- **页面数据**：使用Page对象的data属性存储页面数据
- **本地存储**：使用wx.setStorageSync和wx.getStorageSync进行本地数据存储

### 4.3 API调用

- **微信API**：
  - wx.getSetting：获取用户授权设置
  - wx.authorize：请求用户授权
  - wx.makePhoneCall：拨打电话
  - wx.openLocation：打开地图
  - wx.showModal：显示对话框
  - wx.showToast：显示提示
  - wx.showLoading：显示加载动画

### 4.4 性能优化

- **代码优化**：减少不必要的代码，提高代码执行效率
- **图片优化**：使用适当尺寸的图片，避免过大图片
- **缓存策略**：合理使用缓存，减少重复加载
- **渲染优化**：减少页面渲染时间，提高响应速度

## 5. 开发流程

### 5.1 开发步骤

1. **需求分析**：分析用户需求，确定功能范围
2. **UI/UX设计**：设计界面和交互
3. **代码开发**：分模块进行代码开发
4. **测试**：进行功能测试和用户测试
5. **优化**：根据测试结果进行优化
6. **发布**：提交审核并发布

### 5.2 测试方法

- **功能测试**：测试各个功能模块是否正常工作
- **兼容性测试**：测试在不同设备和微信版本上的表现
- **用户测试**：邀请目标用户进行使用测试
- **性能测试**：测试页面加载速度和响应时间

### 5.3 发布流程

1. **代码审核**：检查代码是否符合微信小程序规范
2. **提交审核**：在微信公众平台提交小程序审核
3. **审核通过**：审核通过后发布上线
4. **版本更新**：根据用户反馈进行版本更新
